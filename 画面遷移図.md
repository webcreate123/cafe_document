## 画面遷移図（ユーザー/店舗/運営）

作成日: 2026-03-25

本図は `cafe_web` / `cafe_admin` の `app/` 配下のページ構成と、`cafe_api` の機能モジュールを対応付けた概略の遷移図である。

### 1. 一般ユーザー画面遷移
```mermaid
stateDiagram-v2
  [*] --> ログイン
  ログイン --> 会員登録: 登録リンク
  会員登録 --> ログイン: 登録完了
  ログイン --> トップ: 認証成功

  トップ --> マップ/検索
  トップ --> 店舗詳細: 新着/ランキング/セール等から選択

  マップ/検索 --> 店舗詳細: 店舗クリック
  店舗詳細 --> お気に入り追加
  店舗詳細 --> 通報/違反報告
  店舗詳細 --> 閲覧履歴: 参照で自動保存

  閲覧履歴 --> 店舗詳細
  お気に入り一覧 --> 店舗詳細: お気に入り店舗クリック

  トップ --> 求人掲示板
  求人掲示板 --> 求人詳細: 求人クリック
  求人詳細 --> 求人お気に入り

  トップ --> 決済管理
  決済管理 --> トップ: プラン切替/支払い完了

  トップ --> お知らせ/通知(画面): 通知一覧表示
  お知らせ/通知(画面) --> 店舗詳細/セール詳細: 通知クリック
```

### 2. 店舗（オーナー/スタッフ）画面遷移
```mermaid
stateDiagram-v2
  [*] --> ログイン
  ログイン --> 店舗掲載申請: 未申請の場合
  店舗掲載申請 --> ログイン: 申請完了（審査待ち）
  ログイン --> 店舗ダッシュボード: 認証成功

  店舗ダッシュボード --> 店舗設定/情報更新
  店舗ダッシュボード --> お知らせ/日記一覧
  店舗ダッシュボード --> 空席状況更新
  店舗ダッシュボード --> セール情報一覧
  店舗ダッシュボード --> 店舗決済管理
  店舗ダッシュボード --> インサイト/分析
  店舗ダッシュボード --> 問い合わせ

  店舗設定/情報更新 --> 店舗ダッシュボード
  お知らせ/日記一覧 --> お知らせ作成/編集
  お知らせ作成/編集 --> お知らせ/日記一覧: 下書き保存/公開
  セール情報一覧 --> セール作成/テンプレ適用
  セール作成/テンプレ適用 --> セール情報一覧

  空席状況更新 --> 店舗ダッシュボード
  店舗決済管理 --> 店舗ダッシュボード: 支払い/領収書処理完了
  問い合わせ --> 店舗ダッシュボード
```

### 3. 運営管理者画面遷移
```mermaid
stateDiagram-v2
  [*] --> 管理ログイン
  管理ログイン --> ダッシュボード: 認証成功

  ダッシュボード --> ユーザー管理
  ダッシュボード --> 店舗承認
  ダッシュボード --> 通報管理
  ダッシュボード --> 決済/売上参照(必要時)
  ダッシュボード --> 分析/イベント(必要時)

  ユーザー管理 --> ダッシュボード: ステータス更新完了
  店舗承認 --> ダッシュボード: 承認/却下/保留反映
  通報管理 --> ダッシュボード: 状態更新/解決処理
```

### 4. 画面とAPI（対応の考え方）
Next.js側の画面は、`cafe_api` の以下prefixと概ね対応する。

- ユーザー/閲覧系: `/stores`, `/favorites`, `/jobs`, `/sales`, `/history`, `/notifications`, `/payments`
- 店舗投稿系: `/announcements`, `/sales`, `/stores`（更新）、`/payments`（店舗）
- 運営管理: `/admin`（統計/ユーザー/店舗/通報の管理）
